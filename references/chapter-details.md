# 章節完整細節

## Ch1：建立基準線

**目標**：在具備「波動一致性」的前提下取得原始訊號的統計底稿。

**核心**：進場時的 ATR（波動率）必須接近。若不同交易的市場環境差異過大（波動率 10 點與 100 點混在一起），MAE/MFE 分佈將失真。

**波動一致性檢驗**：MAE 從 Pips 換成 Daily ATR 單位（如 10 pips → 2 ATRs），讓分佈呈現更明顯的高低波動分離。

**ATR 參數選擇**：
- 短週期（3-4）：即時波動量級
- 中週期（7, 14）：盤段/跨日平均波動
- 長週期（22）：月度波動趨勢
- 選用奇數或質數可避免計算結果太離散

**實現**：
1. 編寫基礎策略並回測
2. 篩選波動度落在特定範圍內的交易樣本
3. 記錄原始損益、MAE、Global MFE
4. 加入延遲進場邏輯作為對照
5. 觀察 MAE, MFE, MFE before MAE 時序圖確認波動穩定性

---

## Ch2：認識 MAE 與 MFE

**記錄欄位**：
- `g_mfe`, `mae`, `mfe`, `mae_lv1`, `mhl`
- `g_mfe_idx`, `mae_idx`, `mfe_idx`, `mae_lv1_idx`, `mhl_idx`（時間索引）
- `excursion_range` = `g_mfe + mae`（進場後完整波動幅度，用於 Base Model 評分）

**時間面 MAE/MFE**：
- Edge Ratio = MFE(t) / MAE(t)（進場後 t 個 Bar 的優勢比）
- 好的訊號：進場後越短時間內 MFE/MAE 比例越高

---

## Ch3：資金管理的核心架構

**核心觀念**：
- 訊號決定方向，資金管理決定結果
- 訊號是「提醒」，不是要求立即在該時間該價格進場
- **主單 + 子單** 結構：主單承載訊號邏輯，子單負責加碼/減碼/對沖

**三階段**：
1. **進場前（Delayed Entry）**：優化進場價格，壓縮初始風險
2. **持倉中（In-Trade Management）**：動態調整停損、加減碼
3. **出場後（Post-Trade Review）**：統計、歸因、參數修正

**帳戶目標 vs 策略目標**：
- 策略：最大化報酬
- 帳戶：階梯式穩定績效（Recovery Factor 回測 8-15，實盤 3-5）

## Ch4：停損決策（SL）

**分析方法**：
1. MAE 對 Profit/Loss 分佈圖（X=MAE, Y=最終損益）
2. MAE 對 Profit/Loss 累計圖
3. 分別觀察：勝手 MAE vs 輸手 MAE、多單 MAE vs 空單 MAE

**實現**：
1. 繪製 MAE CDF，分別標示勝手/輸手
2. 找出 Q1, Q2, Q3 分位數，關注勝手 MAE 集中區域
3. 比較多/空 MAE 分佈差異（為 Ch8 方向過濾準備）
4. 在不同 MAE 分位數設 SL，觀察對整體損益影響

**注意**：收緊 SL 消耗勝率 → 高勝率是前提

---

## Ch5：止盈決策（TP）

**分析方法**：
1. MFE 對 Profit/Loss 分佈圖
2. MFE 對 Profit/Loss 累計圖
3. G_MFE 與 PNL 關係：G_MFE 高但 PNL 偏低 → 出場太晚
4. G_MFE 與 MHL 關係：MHL/G_MFE 小 → 適合追蹤停損；不高 → Lock Profit

**實現**：
1. 繪製勝手/輸手 G_MFE CDF
2. 判斷策略獲利潛力上限
3. 觀察獲利回吐幅度
4. 在不同 G_MFE 分位數設 TP，觀察影響

---

## Ch6：SL/TP Placement Test

**MAE vs MFE 散佈圖**：兩軸比例尺一致

**MFE before MAE 使用時機**：
- MFE 大多在 MAE 之前（波段交易）→ 用 MAE vs MFE before MAE
- MFE 大多在 MAE 之後（趨勢交易）→ 用 MAE vs Global MFE

**集群分佈四型態**：
| 型態 | 特徵 | 對策 |
|------|------|------|
| 向上聚集 | MFE 高，MAE 低 | 專注把握獲利 |
| 向下聚集 | MFE 低，MAE 低 | 收緊 TP，提高周轉率 |
| 向左聚集 | MAE 低，MFE 適中 | 理想分佈，微調即可 |
| 向右聚集 | MAE 高，MFE 適中/低 | 延遲進場或方向過濾 |
| 多個聚集 | 多群 | 分群分別處理 |

**右上角定位**（Q3 以上位置）：
- MFE > MAE 2-5x → 收緊 TP 提早獲利
- MAE > MFE 2-5x → 收緊 SL 減少損失

**L 型反比分佈** → 需要考慮移動停損（Ch9）

**過擬合警告**：SL 切太靠近群集中心 → OOS 崩潰。持續降低 SL + 提高 TP = 降低波動忍耐度 = 本質上過擬合。

## Ch7：延遲進場（Delayed Entry）

### Favorable（限價單 — 壓縮 MAE）

- 參考 **Win Trade MAE 分佈**
- 延遲大小：< MAE_Q1 固定值，或 MAE_Q1 到 (Q1+Q2)/2 嘗試
- **上限**：不超過 MAE_Q3

### Adverse（觸價單 — 過濾失敗訊號）

- 參考 **Loss Trade G_MFE Q3**
- 確認點設在失敗交易通常最高反彈位置之上

### 交互測試

`Sweep L/S, FAV/ADV` — 從每個 base model 產出多組延遲進場模型

### 評估指標

- MAE 是否降低
- 捨棄多少潛在獲利或 G_MFE
- G_MFE / MFE 是否反而成長
- `low g_mfe before mae pct` 是否改善

---

## Ch8：掛單限制與環境過濾

**MAE/MFE 時序圖觀察**：
- 先看 MAE 是否穩定（MAE 穩定，MFE 不穩定沒關係）
- MAE 與 MFE 同時放大/縮小且時間接近 → 分段分析

**過濾條件**：
- 單調進場：限制只在波動特定趨勢時進場
- 週中反轉：W/T/F（週三 GMT 10:00 後到週五 GMT 22:00 前）開倉
- 一月效應、七月效應
- 波動叢聚：波動率變高/低會維持一段時間 → 策略輪換或動態調整 SL/TP 的關鍵時刻

**ATR 飆高注意**：若 ATR 飆高時經常在 F/M/T 進場 → 謹慎考慮過擬合

**GARCH 補充**：
- GARCH 低點捕捉細微時間尺度的波動變化
- 推薦：短週期 ATR + 長週期 ATR + GARCH beta

## Ch9：動態停損

**前提**：需要 MAE vs MFE 的 L 型反比分佈。可不先收緊 SL/TP 就測試。

### BE 設定

- Lose Trade MFE 的 (Q1+Q2)/2 或 Q2 作為觸發價位
- MHL vs G_MFE 直方圖中，高於 Loss Trade G_MFE Q3 可設 BE

### TS 設定

- MHL CDF 作為追蹤幅度參考
- MHL/G_MFE 比例小 → TS；比例大 → Lock Profit

### 判斷依據

- MFE 與 MFE before MAE 重疊性低 → 需要動態停損
- 重疊性高 → 固定 SL/TP

---

## Ch10：加碼策略

### E1 不利方向加碼

- **適用**：勝率高，但輸一次輸很多
- Win Trade MAE Q1~Q2 加碼
- Win Trade MAE Q2~Q3 減碼減回來
- 依賴 Exit Signal 出場

### E2 有利方向加碼

- **適用**：MFE 都較高
- Win Trade MFE Q2~Q3 加碼
- 或 Win/Lose MFE CDF 交叉處加碼（「走到這裡更可能是贏家」）

**每一碼獨立計算風險。**

---

## Ch11：減碼策略

### E3 通則

- 在加碼位置與 TP 位置的 1/2 到 1/3 處減碼提前獲利

### 其他方式

- 依持有時間：超過特定時間後階梯式平倉
- 依獲利幅度：浮盈達 G_MFE 特定分位數時部分平倉

**靈活性**：使用加碼後，減碼可不嚴格參照 MAE/MFE 分析，純粹作為損益最後修飾。

---

## Ch12：資金管理樹（MMT）

**Money Management Tree**：模擬同一訊號下，不同資金管理路徑的結果。

**核心問題**：
- 逆選擇效應：參數難優化時，大多數參數都賠錢 → 被引導到交易數量少的情況 → 用 SQN Value 對抗
- 病態曲率：參數空間太大/維度太高/雜訊太高 → 局部最優

**啟發式搜索四步驟**：精確方法 → 啟發式演算法 → 簡化問題 + 精確方法 → 簡化問題 + 啟發式演算法

**實現**：
1. 多組 SL/TP、延遲進場、加減碼分支化回測
2. 對比 IS vs OOS 表現差異
3. 避免「MDD 變小就是好」的陷阱

---

## Ch13：以資金管理避險

**高階 MAE/MFE**：從單筆交易擴展到帳戶層級，以交易組合整體損益計算。

**機制**：
- 反向單（Hedge Order）：主單大幅 MAE 時開反向單鎖定風險
- 高階反向單：MAE → MFE → MFE 之前的 MAE → 再之前的 MFE... 搭配多組反向單
- 獲利貼補（Win Allowance）：允許一定虧損，只要帳戶仍在獲利範圍
- 組合停損：跨貨幣/跨時框的整體曝險最優停損分配
- 買價/賣價 MAE/MFE：考慮 Bid/Ask 差異影響

**注意**：高階反向單建議僅機構使用。
